# Publicación Gratis

Actualizado para esta entrega con criterio al `20 de abril de 2026`.

## Opción recomendada

`Netlify` es la ruta más rápida para este proyecto porque acepta deploy manual por drag-and-drop y te da una URL `*.netlify.app` al instante.

## Otras opciones sólidas

- `GitHub Pages`: ideal si quieres tener el portfolio versionado en GitHub. Obtienes `usuario.github.io` o `usuario.github.io/repositorio`.
- `Cloudflare Pages`: muy buena opción para sitios estáticos, con dominio `*.pages.dev`.
- `Vercel`: buena si luego quieres previews por rama y flujos más cercanos a front-end moderno. Te da `*.vercel.app`.

## Dominio gratis

La opción gratis más estable es usar el subdominio incluido por la plataforma:

- `tu-sitio.netlify.app`
- `tu-sitio.pages.dev`
- `tu-sitio.vercel.app`
- `usuario.github.io`

Si quieres un dominio más personal sin pagar, hoy las opciones razonables son:

- `is-a.dev`: subdominio gratis orientado a developers y compatible con GitHub Pages, Cloudflare Pages, Vercel y Netlify.
- `js.org`: subdominio gratis, pero orientado principalmente a proyectos relacionados con JavaScript y muy ligado a GitHub Pages.

## Recomendación práctica

1. Publica primero con `Netlify` o `Cloudflare Pages`.
2. Valida que todo funcione.
3. Si todavía quieres un dominio más bonito sin pagar, intenta pedir uno en `is-a.dev`.
4. Si más adelante el portfolio te empieza a generar clientes, compra un dominio propio.

## Nota importante

Los dominios personalizados totalmente gratis suelen ser comunitarios, tardan en aprobarse o son menos confiables que un dominio pagado. Para salir hoy mismo, usa el subdominio del hosting.
